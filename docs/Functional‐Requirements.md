# Функциональные требования 
Система поддерживает выполнение базовых операций (CRUD) для сущностей "Лошадь", "Жокей" и "Забег", а также генерацию статистических отчетов для каждой роли.

## Текстовый сценарий: Распределение призового фонда

**Название:** Расчет и распределение призовых после забега.

**Актор:** Управление (Администратор).

**Предусловие:** Забег завершен, данные о занятых местах внесены в базу.

**Основной поток:**
1. Администратор выбирает опцию "Распределить призовой фонд" для конкретного забега.
2. Система запрашивает общую сумму призового фонда.
3. Администратор вводит сумму.
4. Система вычисляет 50% для первого места, 30% для второго и 20% для третьего.
5. Система зачисляет средства соответствующим участникам и сохраняет результаты в специальную таблицу.
6. Система выдает сообщение об успешном распределении.
