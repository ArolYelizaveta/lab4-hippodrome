# Индивидуальное задание №1: Вариант 19 «Международная компания»

## Постановка задачи

### Вариант 19 из лабораторной работы №3

**«Международная компания»**

База данных содержит следующую информацию о международных компаниях:

- **id** — уникальный идентификатор
- **название** — полное наименование компании
- **интернет сайт** — URL официального сайта
- **адрес главного офиса** — почтовый индекс, страна, область, район, город, улица, дом, квартира
- **год основания** — год создания компании
- **продолжительность пребывания на мировом рынке** — количество лет на рынке
- **количество сотрудников** — общая численность персонала
- **количество филиалов в Европе** — число европейских представительств
- **логотип компании** — изображение (фотография) в формате BLOB

### Функциональные требования к приложению

Приложение должно обеспечивать:

1. **Аутентификацию пользователей** — вход по логину и паролю с разграничением ролей:
   - Администратор — полный доступ ко всем функциям
   - Пользователь — только просмотр и поиск

2. **CRUD операции над компаниями**:
   - Добавление новой компании с возможностью загрузки логотипа
   - Редактирование данных существующей компании
   - Удаление компании по ID
   - Просмотр всех компаний

3. **CRUD операции над категориями компаний**:
   - Категории: Картель, Синдикат, Трест, Концерн
   - Добавление, редактирование, удаление, просмотр категорий

4. **Поиск и фильтрацию**:
   - Поиск компаний по фрагменту названия (LIKE)
   - Фильтрация компаний по категории

5. **Экспорт данных**:
   - Сохранение логотипа компании в файл
   - Экспорт базы данных в форматы .sql и .csv

6. **Демонстрацию режимов работы с БД**:
   - Режим autocommit (построчное выполнение)
   - Режим transaction (групповое выполнение с BEGIN/COMMIT)

### Текстовый сценарий: UC3 «Добавить компанию»

Название: Добавить международную компанию
Актер: Администратор
Предусловие: Пользователь авторизован как администратор
Постусловие: Компания добавлена в базу данных

Основной поток:
1. Администратор выбирает пункт меню "Добавить компанию"
2. Система запрашивает название компании
3. Администратор вводит название
4. Система запрашивает website
5. Администратор вводит website
6. Система запрашивает адрес главного офиса
7. Администратор вводит адрес
8. Система запрашивает год основания
9. Администратор вводит год основания
10. Система запрашивает количество лет на рынке
11. Администратор вводит количество лет
12. Система запрашивает количество сотрудников
13. Администратор вводит количество сотрудников
14. Система запрашивает количество филиалов в Европе
15. Администратор вводит количество филиалов
16. Система запрашивает ID категории
17. Администратор вводит ID категории
18. Система запрашивает путь к файлу логотипа
19. Администратор вводит путь к файлу
20. Система сохраняет компанию в БД
21. Система выводит сообщение об успехе

Альтернативный поток (ошибка валидации):
- Если поле не заполнено или ID категории не существует
- Система выводит сообщение об ошибке
- Возврат к шагу 2
