# Дополнительная спецификация

## Нефункциональные требования
* **Надёжность:** Система должна корректно обрабатывать неверный ввод пользователя (например, ввод текста вместо числа) без аварийного завершения работы. При отсутствии соответствия кличек лошадей и фамилий жокеев внесение информации в таблицу забегов должно блокироваться.
* **Удобство использования (Usability):** Консольный интерфейс должен иметь интуитивно понятное текстовое меню с четкой нумерацией пунктов.
* **Производительность:** Время отклика базы данных SQLite при генерации статистических отчетов (например, выборка лучшей лошади) не должно превышать 2 секунд.
* **Сопровождаемость:** Исходный код должен быть прокомментирован, а архитектура должна позволять легко добавлять новые сущности (например, "Тренер") в будущем.
