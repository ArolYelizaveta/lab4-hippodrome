# Система управления ипподромом

## Требования

- Linux / macOS / Windows (с MinGW или WSL)
- Компилятор g++ с поддержкой C++11
- Библиотека SQLite3

## Установка SQLite3

Ubuntu/Debian:
```
sudo apt-get install sqlite3 libsqlite3-dev
```

CentOS/RHEL/Fedora:
```
sudo yum install sqlite sqlite-devel
```

macOS:
```
brew install sqlite3
```

## Сборка

Скопируйте исходный код и выполните:

```
make
```

## Запуск

```
make run
```

Или:

```
./bin/hippodrome
```

## Очистка файлов сборки

```
make clean
```

## Создание базы данных

В корне проекта находится файл `hippodrome.sql`. Выполните следующую команду для создания базы данных:

```
sqlite3 hippodrome.db < hippodrome.sql
```

## Вход в систему

Поддерживаются две роли пользователей:

- Владелец - управление лошадьми, просмотр результатов забегов
- Жокей - просмотр личной истории забегов, участие в забегах

Используйте зарегистрированные имя пользователя и пароль из таблицы Users.

## Возможности по ролям

### Владелец
1. Просмотр моих лошадей и забегов
2. Информация о лучшей лошади
3. Забеги за указанный период
4. Добавить новую лошадь
5. Обновить данные лошади
6. Удалить лошадь
7. Добавить забег
8. Распределить призовые

### Жокей
1. Мои забеги
2. Забеги за указанный период
3. Информация о самом активном жокее
4. Добавить забег
5. Распределить призовые

## Формат даты

Используйте формат ГГГГ-ММ-ДД для всех дат (например, 2024-01-15).
